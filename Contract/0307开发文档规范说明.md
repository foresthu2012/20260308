## 二、实现的功能点清单

### 2.1 后台管理端（admin）

| 功能点 | 说明 | 对应文件 |
|--------|------|----------|
| 合同列表 | 分页列表、按标题/会员/状态搜索、查看/编辑/删除 | `admin/views/contract/contract/index.vue` |
| 添加合同 | 填写标题、选择会员、关联订单、上传合同文件 | `admin/views/contract/contract/edit.vue` |
| 编辑合同 | 待签署状态可编辑，已签署仅可查看 | `admin/views/contract/contract/edit.vue` |
| 删除合同 | 软删除合同记录 | `admin/views/contract/contract/index.vue` |
| 合同预览 | PDF/图片格式预览，支持缩放、全屏 | `admin/views/contract/contract/FilePreviewDialog.vue` |
| 会员选择 | 弹窗选择会员，支持昵称/手机号搜索 | `admin/views/contract/contract/member_select_popup.vue` |
| 订单选择 | 弹窗选择订单，按 member_id 筛选该会员订单 | `admin/views/contract/contract/order_select_popup.vue` |

### 2.2 会员端 API（app/api）

| 功能点 | 说明 | 对应接口 |
|--------|------|----------|
| 合同列表 | 分页获取当前会员的合同，支持按状态筛选 | `GET /Contract/contract` |
| 合同详情 | 获取单条合同详情 | `GET /Contract/contract/:id` |
| 微信身份签署 | 用户授权后使用微信 openid 生成电子签章完成签署 | `POST /Contract/contract/confirm_sign/:id` |

### 2.3 移动端（uni-app）

| 功能点 | 说明 | 对应页面 |
|--------|------|----------|
| 我的合同 | 合同列表，待签署/已签署 Tab 切换，下拉刷新、上拉加载 | `uni-app/pages/contract/list.vue` |
| 合同预览 | 点击合同直接全屏预览（图片/PDF），适配手机屏、安全区 | `uni-app/pages/contract/list.vue` |
| 合同详情 | 查看合同、预览文件、微信授权并签署 | `uni-app/pages/contract/detail.vue` |
| 微信身份签署 | 授权后使用微信身份生成电子签章，签署成功展示签署方式 | `uni-app/pages/contract/detail.vue` |

### 2.4 后端业务（app/service）

| 功能点 | 说明 | 对应文件 |
|--------|------|----------|
| 合同 CRUD | 增删改查、分页、关联会员/订单 | `app/service/admin/contract/ContractService.php` |
| 会员合同列表/详情 | 按 member_id 过滤，返回 file_path、sign_image 等 | `app/service/api/contract/ContractService.php` |
| 微信身份签署 | 调用微信 jscode2session 获取 openid，生成印章、合成合同图 | `app/service/api/contract/ContractService.php` |

## 三、强制规范对照

### 3.1 钩子函数

| 规范要求 | 本插件使用情况 | 说明 |
|----------|----------------|------|
| 仅允许使用规范内钩子（appInit、adminInit、memberRegister、memberLogin、orderPay 等） | **未使用任何钩子** | `app/event.php` 中 `listen`、`subscribe` 均为空，插件未注册任何事件监听 |

```php
// app/event.php 当前内容
return [
    'listen' => [],
    'subscribe' => [],
];
```

### 3.2 路由配置

| 类型 | 规范要求 | 本插件实现 | 对应功能 |
|------|----------|------------|----------|
| 菜单路由 | 必须以 `addon/插件标识/` 开头 | ✅ `/addon/Contract/pages/contract/list` | 会员端「我的合同」入口 |
| 会员菜单 | - | `app/dict/menu/member.php` | 我的合同（router_path） |
| 自定义链接 | - | `app/dict/diy/links.php` | 我的合同（diy_link） |
| 后台接口 | 统一放在 `app/adminapi/route/route.php` | ✅ | 见下表 |
| 前台接口 | - | `app/api/route/route.php` | 见下表 |

**后台接口路由（adminapi）**

| 路由 | 方法 | 控制器方法 | 功能 |
|------|------|------------|------|
| `Contract/contract` | GET | Contract@lists | 合同列表 |
| `Contract/contract/:id` | GET | Contract@info | 合同详情 |
| `Contract/contract` | POST | Contract@add | 添加合同 |
| `Contract/contract/:id` | PUT | Contract@edit | 编辑合同 |
| `Contract/contract/:id` | DELETE | Contract@delete | 删除合同 |
| `Contract/contract/:id/view` | GET | Contract@viewSigned | 查看已签署合同 |
| `Contract/Shop/order/lists` | GET | Order@lists | 商城订单列表（关联用） |

**前台接口路由（api）**

| 路由 | 方法 | 控制器方法 | 功能 |
|------|------|------------|------|
| `Contract/contract` | GET | Contract@lists | 会员合同列表 |
| `Contract/contract/:id` | GET | Contract@info | 会员合同详情 |
| `Contract/contract/sign/:id` | POST | Contract@sign | 手写签名上传（保留，当前未用） |
| `Contract/contract/confirm_sign/:id` | POST | Contract@confirmSign | 微信身份签署 |

**uni-app 页面路由**

| 路径 | 标题 | 功能 |
|------|------|------|
| `addon/Contract/pages/contract/list` | 我的合同 | 合同列表、直接预览 |
| `addon/Contract/pages/contract/detail` | 合同详情 | 合同详情、微信身份签署 |

### 3.3 文件上传组件

| 规范要求 | 本插件使用情况 | 实现功能 |
|----------|----------------|----------|
| 必须使用官方上传组件（image-upload / images-upload / file-upload） | ✅ 使用 `upload-file` | 后台添加/编辑合同时上传合同文件（PDF、JPG、PNG） |
| 禁止自定义上传逻辑 | ✅ 未自定义 | 合同文件通过 `upload-file` 上传至系统附件目录 |

**使用位置**：`admin/views/contract/contract/edit.vue`

```vue
<upload-file v-model="formData.file_path" :limit="1" accept=".pdf,.jpg,.jpeg,.png" />
```

## 五、数据库与模型

### 5.1 插件自建表

| 表名主体 | 实际表名（框架自动加前缀） | 说明 |
|----------|----------------------------|------|
| addon_contract_contract | ns_addon_contract_contract | 合同主表 |
| addon_contract_signature_record | ns_addon_contract_signature_record | 签署记录表（openid、签章路径等） |

### 5.2 系统核心表引用

| 表名主体 | 用途 |
|----------|------|
| member | 合同关联会员，会员选择 |
| shop_order | 合同关联订单，订单选择（按 member_id 筛选） |
| config | 读取 WECHAT_MINIAPP_CONFIG 用于微信身份签署 |

### 5.3 模型规范遵守情况

- ✅ 仅使用 `protected $name = '表名主体'`，无 `protected $table` 硬编码
- ✅ 插件表命名：`addon_contract_*`（符合 addon_插件标识_表名）
- ✅ 系统表引用：`shop_order`、`member` 等仅写表名主体
