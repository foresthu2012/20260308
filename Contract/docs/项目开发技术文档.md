## 二、功能整体说明

### 2.1 业务目标

Contract 插件为电商平台提供完整的**电子合同签署解决方案**，实现合同上传、会员签署、电子签章生成、合同管理全流程。

**核心业务场景：**
1. 商家在后台上传合同文件（PDF/图片），选择签约会员，关联订单
2. 会员在移动端查看合同，使用微信授权方式签署（电子签章）
3. 系统自动生成电子签章并合成到合同文件，标记合同为已签署
4. 商家可查看、管理所有合同记录

### 2.2 解决的问题

| 问题 | 解决方案 |
|------|----------|
| 合同签署流程繁琐，需要线下签署或邮寄 | 在线电子签署，一键完成 |
| 签署身份难以追溯，存在法律风险 | 微信身份授权 + 电子签章，符合《电子签名法》 |
| 合同文件管理混乱，难以查询归档 | 系统化管理，支持搜索、预览、下载 |
| 合同版本难以控制，易被篡改 | 电子签章 + 时间戳，防篡改可追溯 |

### 2.3 功能范围

#### 后台管理端（admin）

| 功能模块 | 功能点 |
|----------|--------|
| 合同管理 | 合同列表（分页、搜索）、添加合同、编辑合同、删除合同、合同预览 |
| 会员选择 | 弹窗选择会员，支持昵称/手机号搜索 |
| 订单关联 | 弹窗选择订单，按会员筛选 |

| 合同详情 | 获取单条合同详情 |
| 微信授权签署 | 使用微信身份生成电子签章并完成签署 |

#### 移动端（uni-app）

## 六、路由配置说明

### 6.1 后台路由

#### 6.1.1 后台 API 路由

**文件位置**：`app/adminapi/route/route.php`

**路由定义**：
```php
use addon\Contract\app\adminapi\controller\contract\Contract;

// 合同列表
Route::get('Contract/contract', 'addon\Contract\app\adminapi\controller\contract\Contract@lists');

// 合同详情
Route::get('Contract/contract/:id', 'addon\Contract\app\adminapi\controller\contract\Contract@info');

// 添加合同
Route::post('Contract/contract', 'addon\Contract\app\adminapi\controller\contract\Contract@add');

// 编辑合同
Route::put('Contract/contract/:id', 'addon\Contract\app\adminapi\controller\contract\Contract@edit');

// 删除合同
Route::delete('Contract/contract/:id', 'addon\Contract\app\adminapi\controller\contract\Contract@delete');
```

#### 6.1.2 路由参数

| 路由 | 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|------|
| `Contract/contract` | `page` | int | 否 | 页码，默认 1 |
| `Contract/contract` | `limit` | int | 否 | 每页数量，默认 10 |
| `Contract/contract` | `title` | string | 否 | 合同标题搜索 |
| `Contract/contract` | `status` | int | 否 | 状态筛选（0 待签署，1 已签署） |
| `Contract/contract` | `member_id` | int | 否 | 会员 ID |
| `Contract/contract` | `member_keyword` | string | 否 | 会员关键词（昵称/手机号） |
| `Contract/contract/:id` | `id` | int | 是 | 合同 ID |
| `Contract/contract` (POST/PUT) | `title` | string | 是 | 合同标题 |
| `Contract/contract` (POST/PUT) | `member_id` | int | 是 | 会员 ID |
| `Contract/contract` (POST/PUT) | `file_path` | string | 是 | 合同文件路径 |
| `Contract/contract` (POST/PUT) | `order_id` | int | 否 | 关联订单 ID |

#### 6.1.3 路由守卫与权限控制

后台路由通过 Niushop 框架的中间件自动处理：
- **身份认证**：自动验证管理员登录状态
- **站点隔离**：自动过滤当前站点数据（`site_id`）
- **权限控制**：通过系统权限中心配置，无需在代码中硬编码

### 7.2 功能二：电子签章生成与合成（核心）

#### 7.2.1 实现思路

使用 PHP GD 库生成电子签章（红色圆形印章），然后将签章合成到合同文件右下角，最终生成已签署的合同图片。

#### 7.2.2 关键技术手段

1. **电子签章生成**：使用 `imagecreatetruecolor()` 创建画布，绘制多层圆环、文字
2. **微信身份验证**：调用微信 `jscode2session` API 获取 openid
3. **图片合成**：使用 `imagecopyresampled()` 将签章合成到合同图片


```php
private function generateStamp($openid)
{
    $width = 320;
    $height = 320;
    $img = imagecreatetruecolor($width, $height);
    
    // 颜色定义
    $transparent = imagecolorallocatealpha($img, 0, 0, 0, 127);
    $red = imagecolorallocate($img, 220, 20, 60);
    $darkRed = imagecolorallocate($img, 178, 34, 34);
    $white = imagecolorallocate($img, 255, 255, 255);
    
    imagefill($img, 0, 0, $transparent);
    imagesavealpha($img, true);
    
    $centerX = $width / 2;
    $centerY = $height / 2;
    
    // 绘制外圈圆环
    $radius = 150;
    imageellipse($img, $centerX, $centerY, $radius * 2, $radius * 2, $red);
    imageellipse($img, $centerX, $centerY, $radius * 2 - 4, $radius * 2 - 4, $red);
    imageellipse($img, $centerX, $centerY, $radius * 2 - 8, $radius * 2 - 8, $darkRed);
    
    // 绘制内圈
    $innerRadius = 140;
    imageellipse($img, $centerX, $centerY, $innerRadius * 2, $innerRadius * 2, $darkRed);
    
    // 绘制虚线
    for ($angle = 0; $angle < 360; $angle += 10) {
        if ($angle % 20 !== 0) {
            $rad = deg2rad($angle);
            $x1 = $centerX + cos($rad) * ($innerRadius - 8);
            $y1 = $centerY + sin($rad) * ($innerRadius - 8);
            $x2 = $centerX + cos($rad) * ($innerRadius - 4);
            $y2 = $centerY + sin($rad) * ($innerRadius - 4);
            imageline($img, $x1, $y1, $x2, $y2, $red);
        }
    }
    
    // 绘制文字
    $text = "已签署";
    $font = 5;
    $textWidth = imagefontwidth($font) * mb_strlen($text);
    imagestring($img, $font, ($width - $textWidth) / 2, $centerY - 45, $text, $red);
    
    $text = "CONFIRMED";
    $textWidth = imagefontwidth($font) * strlen($text);
    imagestring($img, $font, ($width - $textWidth) / 2, $centerY - 15, $text, $darkRed);
    
    $maskedOpenid = 'ID: ' . substr($openid, -8);
    $textWidth = imagefontwidth($font) * strlen($maskedOpenid);
    imagestring($img, $font, ($width - $textWidth) / 2, $centerY + 15, $maskedOpenid, $darkRed);
    
    $date = date('Y-m-d H:i');
    $textWidth = imagefontwidth($font) * strlen($date);
    imagestring($img, $font, ($width - $textWidth) / 2, $centerY + 40, $date, $darkRed);
    
    // 保存签章图片
    $fileName = 'stamp_' . time() . '_' . rand(1000, 9999) . '.png';
    $uploadDir = 'upload/contract_sign/';
    $savePath = app()->getRootPath() . 'public/' . $uploadDir;
    
    if (!is_dir($savePath)) {
        mkdir($savePath, 0755, true);
    }
    
    imagepng($img, $savePath . $fileName);
    imagedestroy($img);
    
    return $uploadDir . $fileName;
}
```

**签章合成**（`app/service/api/contract/ContractService.php`）：
```php
private function compositeStampToContract($contract, $stampPath)
{
    $rootPath = app()->getRootPath();
    $publicPath = $rootPath . 'public/';
    
    // 加载合同原图
    $bgPath = $publicPath . $contract['file_path'];
    $imageInfo = getimagesize($bgPath);
    $bgMime = $imageInfo['mime'];
    
    $bgImg = $this->loadImage($bgPath, $bgMime);
    
    // 加载签章图片
    $stampImg = imagecreatefrompng($publicPath . $stampPath);
    
    // 计算尺寸和位置
    $bgWidth = imagesx($bgImg);
    $bgHeight = imagesy($bgImg);
    $stampWidth = imagesx($stampImg);
    $stampHeight = imagesy($stampImg);
    
    // 签章大小为文档宽度的 22%
    $targetStampWidth = $bgWidth * 0.22;
    $ratio = $targetStampWidth / $stampWidth;
    $targetStampHeight = $stampHeight * $ratio;
    
    // 位置：右下角，边距 4%
    $padding = $bgWidth * 0.04;
    $destX = $bgWidth - $targetStampWidth - $padding;
    $destY = $bgHeight - $targetStampHeight - $padding;
    
    // 合成
    imagealphablending($bgImg, true);
    imagesavealpha($bgImg, true);
    imagealphablending($stampImg, true);
    imagesavealpha($stampImg, true);
    
    imagecopyresampled(
        $bgImg, $stampImg,
        $destX, $destY, 0, 0,
        $targetStampWidth, $targetStampHeight,
        $stampWidth, $stampHeight
    );
    
    // 保存最终合同
    $fileName = 'signed_contract_' . time() . '_' . rand(1000, 9999) . '.jpg';
    $uploadDir = 'upload/contract_sign/';
    $saveDir = $publicPath . $uploadDir;
    
    if (!is_dir($saveDir)) {
        mkdir($saveDir, 0755, true);
    }
    
    imagejpeg($bgImg, $saveDir . $fileName, 90);
    
    // 清理资源
    imagedestroy($bgImg);
    imagedestroy($stampImg);
    
    return $uploadDir . $fileName;
}
```

- **微信身份验证**：通过 openid 绑定签署身份，确保签署可追溯、防篡改

```typescript
export function img(path: string): string {
    if (!path) return ''
    
    // 如果已经是完整URL，直接返回
    if (path.startsWith('http://') || path.startsWith('https://')) {
        return path
    }
    
    // 获取基础URL
    let baseUrl = ''
    if (typeof import.meta !== 'undefined' && import.meta.env?.VITE_BASE_URL) {
        baseUrl = import.meta.env.VITE_BASE_URL
    } else if (typeof window !== 'undefined' && window.location) {
        baseUrl = window.location.origin
    } else {
        baseUrl = 'http://localhost:8080'
    }
    
    // 避免重复添加 /upload/ 前缀
    if (path.startsWith('/')) {
        return baseUrl + path
    } else if (path.startsWith('upload/')) {
        return baseUrl + '/' + path
    } else {
        return baseUrl + '/upload/' + path
    }
}
```

**设计原因**：
- 统一图片路径处理逻辑
- 避免重复添加 `/upload/` 前缀
- 多级 baseUrl 获取策略（环境变量 > window.location > 默认值）

```php
public function parseFileExt(string $filePath): string
{
    if (empty($filePath)) {
        return '';
    }
    $pathInfo = pathinfo($filePath);
    return isset($pathInfo['extension']) ? strtolower($pathInfo['extension']) : '';
}

public function autoParseFileExt(array $data): array
{
    if (!empty($data['file_path'])) {
        $data['file_ext'] = $this->parseFileExt($data['file_path']);
    }
    return $data;
}
```

## 十一、注意事项、常见问题、坑点

### 11.1 开发注意点

| 注意点 | 说明 | 影响 |
|--------|------|------|
| **数据归属** | 后端必须验证 `member_id` 和 `site_id` | 防止数据越权访问 |
| **文件扩展名** | 必须自动解析 `file_ext`，避免手动输入 | 确保数据准确性 |
| **状态控制** | 已签署合同不可编辑/删除 | 防止篡改已签署合同 |
| **订单关联** | 订单必须属于所选会员（`order.member_id == contract.member_id`） | 确保数据一致性 |
| **微信配置** | 必须配置 `WECHAT_MINIAPP_CONFIG`（appid/app_secret） | 否则无法使用微信授权签署 |
| **图片路径处理** | 使用 `img()` 函数统一处理，避免重复添加 `/upload/` 前缀 | 确保图片正确显示 |
| **GD 库依赖** | 服务器必须安装 GD 库并支持图片处理 | 否则无法生成电子签章 |
| **签署记录** | 必须保存到 `signature_record` 表，防重复签署 | 确保签署唯一性 |

### 11.2 容易出错的逻辑

| 场景 | 易错点 | 解决方案 |
|------|--------|----------|
| **会员变更时订单清空** | 会员变更后未清空已选订单，导致数据不一致 | 在 `handleMemberSelect()` 中强制清空订单 |
| **文件扩展名解析** | 手动输入容易出错 | 自动解析 `file_ext`，禁止手动输入 |
| **图片路径处理** | 重复添加 `/upload/` 前缀导致 404 | 使用 `img()` 函数智能判断并处理 |
| **PDF 预览** | 直接显示 PDF 无法预览 | 使用 `uni.openDocument()` 系统阅读器 |
| **微信授权失败** | 未配置 `WECHAT_MINIAPP_CONFIG` | 提示用户配置，并返回明确错误码 |
| **重复签署** | 同一用户多次签署同一合同 | 通过 `openid + contract_id` 组合键防止 |
| **签章合成失败** | 图片格式不支持导致合成失败 | 使用 `loadImage()` 统一加载，支持多种格式 |
| **数据过滤** | 未过滤 `site_id` 导致多店铺数据混乱 | 后端自动过滤，强制使用当前站点数据 |

## 十二、文档总结

### 12.1 核心技术点总结

| 技术点 | 实现方式 | 价值 |
|--------|----------|------|
| **电子签章生成** | PHP GD 库绘制多层圆环、文字、时间戳 | 符合《电子签名法》，具有法律效力 |
| **微信身份验证** | 调用微信 jscode2session 获取 openid | 确保签署身份可追溯、防篡改 |
| **图片合成** | 使用 imagecopyresampled() 合成签章到合同 | 生成专业已签署合同 |
| **全屏预览** | Vue 全屏遮罩 + scroll-view 实现沉浸式预览 | 提升用户体验 |
| **防重复签署** | openid + contract_id 组合键防止 | 确保签署唯一性 |
| **数据归属控制** | 自动过滤 member_id、site_id | 防止数据越权访问 |

### 12.2 业务流程总结

```
商家操作流程：
1. 上传合同文件
2. 选择签约会员
3. 可选：关联订单
4. 提交保存

会员操作流程：
1. 查看我的合同（待签署/已签署）
2. 点击合同全屏预览
3. 勾选协议，点击"微信授权并签署"
4. 系统生成电子签章并合成到合同
5. 签署完成，合同自动生效

签署技术流程：
1. 调用 uni.login 获取微信 code
2. 后端调用微信 jscode2session 获取 openid
3. 生成电子签章（320x320px，红色圆形）
4. 合成签章到合同右下角（占文档宽度 22%）
5. 保存签署记录和最终合同文件
6. 更新合同状态为已签署
```
