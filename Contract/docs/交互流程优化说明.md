# 合同签署交互流程优化说明

## 优化目标

基于用户反馈，优化合同签署的交互流程，实现：
1. **一键预览**：用户点击合同卡片即可直接进入全屏预览模式
2. **便捷签署**：预览页面底部添加"微信授权签署"按钮，操作路径简洁直观

## 优化前的问题

从截图可以看出，原有界面存在以下问题：

### 1. 合同详情页（第一张图）
- ❌ 显示"请在下方区域签名"的手写签名区域（已废弃功能）
- ❌ "重写"和"提交签名"按钮（旧版手写签名按钮）
- ❌ 需要点击"点击查看合同"才能预览

### 2. 合同列表页（第二张图）
- ✅ 已有"待签署"和"已签署"标签
- ✅ 显示合同标题和创建时间
- ⚠️ 需要优化点击后的交互流程

## 交互细节

### 1. 合同卡片点击区域

| 区域 | 点击效果 |
|------|---------|
| 卡片头部（标题+状态） | 进入全屏预览 |
| 卡片主体（订单信息+时间） | 进入全屏预览 |
| 卡片底部按钮 | 进入全屏预览 |

### 2. 全屏预览模式

**图片预览：**
- 宽度适配屏幕
- 支持上下滑动查看长图
- 长按可保存图片

**PDF预览：**
- 显示"打开PDF"和"下载"按钮
- 点击"打开PDF"调用系统阅读器

**底部操作栏（仅待签署）：**
- 协议勾选框（必须勾选才能签署）
- "立即签署"按钮（绿色渐变）
- 未勾选时按钮置灰

### 3. 签署流程

```
1. 用户点击合同 → 进入全屏预览
2. 用户阅读合同内容
3. 用户勾选"我已阅读并同意..."
4. 用户点击"立即签署"
5. 弹出确认对话框
6. 调用微信登录获取code
7. 后端生成电子签章并合成合同
8. 显示"签署成功"提示
9. 关闭预览，刷新列表
```

## 技术实现

### 图片路径处理
统一使用 `img()` 函数处理图片路径：
```typescript
const img = (path: string) => {
    if (!path) return ''
    if (path.startsWith('http')) return path
    const baseUrl = import.meta.env.VITE_BASE_URL || 'http://localhost:8080'
    if (path.startsWith('/')) {
        return baseUrl + path
    } else if (path.startsWith('upload/')) {
        return baseUrl + '/' + path
    } else {
        return baseUrl + '/upload/' + path
    }
}
```
