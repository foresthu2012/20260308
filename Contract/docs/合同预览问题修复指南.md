# 合同预览问题修复指南

## 问题描述
点击"合同文件:点击查看合同"无法正常预览合同内容

## 问题原因

### 1. 图片路径处理逻辑错误

**原代码问题** (list.vue:150-155):
```typescript
const img = (path: string) => {
    if (!path) return ''
    if (path.startsWith('http')) return path
    const baseUrl = import.meta.env.VITE_BASE_URL || ''
    return baseUrl + '/upload/' + path  // 强制添加 /upload/ 前缀
}
```

**问题分析**:
- 数据库中 `file_path` 字段存储的是完整相对路径,如 `upload/contract/xxx.jpg` 或 `upload/contract_sign/signed_contract_xxx.jpg`
- 代码强制在路径前添加 `/upload/`,导致路径变成 `/upload/upload/contract/xxx.jpg`
- 结果: 文件404,无法加载

### 2. 环境变量未配置

`import.meta.env.VITE_BASE_URL` 可能未配置,导致 baseUrl 为空字符串。

### 3. common.ts 中的实现与 list.vue 不一致

common.ts 使用 `window.location.origin` 而 list.vue 使用 `import.meta.env.VITE_BASE_URL`

## 解决方案

### 已修复文件

#### 1. `/uni-app/pages/contract/list.vue` (Line 149-165)

**修改后的代码**:
```typescript
// 图片处理
const img = (path: string) => {
    if (!path) return ''
    // 如果已经是完整URL，直接返回
    if (path.startsWith('http://') || path.startsWith('https://')) {
        return path
    }
    // 如果是相对路径，添加基础URL
    const baseUrl = import.meta.env.VITE_BASE_URL || 'http://localhost:8080'
    // 避免重复添加 /upload/ 前缀
    if (path.startsWith('/')) {
        return baseUrl + path
    } else if (path.startsWith('upload/')) {
        return baseUrl + '/' + path
    } else {
        return baseUrl + '/upload/' + path
    }
}
```

**改进点**:
- 增加对完整URL的判断 (http:// / https://)
- 检查路径是否已包含 `upload/` 前缀,避免重复添加
- 提供默认baseUrl: `http://localhost:8080`

#### 2. `/uni-app/utils/common.ts` (Line 6-38)

**修改后的代码**:
```typescript
/**
 * 图片路径处理函数
 * @param path 图片路径
 * @returns完整的图片URL
 */
export function img(path: string): string {
    if (!path) return ''

    // 如果已经是完整URL，直接返回
    if (path.startsWith('http://') || path.startsWith('https://')) {
        return path
    }

    // 获取基础URL
    let baseUrl = ''
    // 优先使用环境变量
    if (typeof import.meta !== 'undefined' && import.meta.env?.VITE_BASE_URL) {
        baseUrl = import.meta.env.VITE_BASE_URL
    }
    // 其次使用 window.location
    else if (typeof window !== 'undefined' && window.location) {
        baseUrl = window.location.origin
    }
    // 最后使用默认值
    else {
        baseUrl = 'http://localhost:8080'
    }

    // 避免重复添加 /upload/ 前缀
    if (path.startsWith('/')) {
        return baseUrl + path
    } else if (path.startsWith('upload/')) {
        return baseUrl + '/' + path
    } else {
        return baseUrl + '/upload/' + path
    }
}
```

**改进点**:
- 统一图片路径处理逻辑
- 多级baseUrl获取策略: 环境变量 > window.location > 默认值
- 避免重复添加 `upload/` 前缀

## 配置说明

### 配置 API 基础URL

根据你的项目部署环境,需要配置正确的 `VITE_BASE_URL`:

#### 方法1: 环境变量文件 (推荐)

在项目根目录创建 `.env.development` 或 `.env.production` 文件:

```bash
# 开发环境 (.env.development)
VITE_BASE_URL=http://localhost:8080

# 或你的实际开发服务器地址
VITE_BASE_URL=http://192.168.1.100:8080

# 生产环境 (.env.production)
VITE_BASE_URL=https://your-domain.com
```

#### 方法2: manifest.json 配置

修改 `/uni-app/manifest.json`,在 `mp-weixin` 中添加:

```json
{
  "mp-weixin": {
    "appid": "你的小程序appid",
    "setting": {
      "urlCheck": false,
      "es6": true,
      "postcss": true,
      "minified": true
    },
    "usingComponents": true,
    "networkTimeout": {
      "request": 60000
    }
  }
}
```

#### 方法3: 运行时配置 (临时测试)

在页面加载时动态设置:

```typescript
// 在 App.vue 或 main.ts 中
if (typeof window !== 'undefined') {
  window.location.hostname
}
```

## 数据库字段说明

### contract 表

| 字段名 | 类型 | 说明 | 示例值 |
|--------|------|------|--------|
| file_path | varchar | 原始合同文件路径 | `upload/contract/xxx.jpg` |
| sign_image | varchar | 已签署合同文件路径 | `upload/contract_sign/signed_contract_xxx.jpg` |
| status | tinyint | 签署状态 | 0=待签署, 1=已签署 |

### 路径格式规范

- ✅ 正确格式: `upload/contract/file.jpg`
- ✅ 正确格式: `upload/contract_sign/signed_xxx.jpg`
- ❌ 错误格式: `/upload/contract/file.jpg` (开头的斜杠)
- ❌ 错误格式: `contract/file.jpg` (缺少 upload/)

## 常见问题

### Q1: 图片加载失败,显示404

**原因**: 路径配置错误

**解决**:
1. 检查数据库中 `file_path` 字段的值
2. 确认路径格式为 `upload/xxx/xxx.jpg`
3. 检查 `VITE_BASE_URL` 配置是否正确
4. 在浏览器中直接访问生成的URL测试

### Q2: 微信小程序中图片无法加载

**原因**: 微信小程序域名白名单未配置

**解决**:
1. 登录微信公众平台
2. 进入"开发 > 开发管理 > 开发设置"
3. 配置服务器域名白名单,将你的域名添加到:
   - uploadFile 合法域名
   - downloadFile 合法域名

### Q3: 已签署合同看不到签章

**原因**: `sign_image` 字段未正确更新

**解决**:
1. 检查后端签署逻辑是否正常执行
2. 查看数据库 `sign_image` 字段是否有值
3. 确认签章图片文件是否存在于服务器

### Q4: PDF合同无法预览

**原因**: PDF需要特殊处理

**解决**:
当前代码已实现PDF预览功能,点击"打开PDF"按钮会:
1. 下载PDF文件到本地
2. 调用系统工具打开
